3. 创建程序

创建一个 Quarkus 项目，最简单方法是打开终端并运行以下命令：

对与Linux和Mac用户

CLI
quarkus create app org.acme:getting-started \
    --extension=resteasy
cd getting-started

要创建 Gradle 项目，请添加 --gradle 或 --gradle-kotlin-dsl 选项。

有关如何安装和使用 Quarkus CLI 的更多信息，请参阅 Quarkus CLI 教程。

Maven
mvn io.quarkus.platform:quarkus-maven-plugin:999-SNAPSHOT:create \
    -DprojectGroupId=org.acme \
    -DprojectArtifactId=getting-started \
    -Dextensions="resteasy"
cd getting-started

要创建 Gradle 项目请添加 -DbuildTool=gradle 或 -DbuildTool=gradle-kotlin-dsl 选项

对与Windows用户

  • 如果使用CMD，不要使用反斜杠 \ 请将内容放在同一行内。

  • 如果使用Powershell，请将 -D 的参数信息用双引号将其包裹，例如 "-DprojectArtifactId=getting-started"。

项目会生成在 ./getting-started 目录中，该项目包含如下内容：

  • 一个Maven架构的工程

  • 提供 /hello 端点的资源类 org.acme.GreetingResource

  • 一个与端点相关的单元测试

  • 启动应用程序后，在 http:localhost:8080 上访问的登录页面

  • 在 src/main/docker 目录下提供进行 native 或者 jvm 部署的 Dockerfile 示例文件

  • 应用程序配置文件

项目创建完成后查看 pom.xml 文件。 您会发现导入了 Quarkus BOM 依赖，它允许省略不同Quarkus依赖向上的版本。 此外，您还可以看到 quarkus-maven-plugin 插件，该插件负责提供打包程序以及开发模式。

    <dependencies>

            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>

<build>
    <plugins>
        <plugin>
            <groupId>${quarkus.platform.group-id}</groupId>

        </plugin>

</build>

  • Quarkus Gradle 插件

  • enforcedPlatform 指令导入Quarkus BOM

3.1. JAX-RS 资源

在项目创建过程中， src/main/java/org/acme/GreetingResource.java 文件已经被创建，文件内容如下：

package org.acme;

import javax.ws.rs.GET;
import javax.ws.rs.Path;
import javax.ws.rs.Produces;
import javax.ws.rs.core.MediaType;

@Path("/hello")
public class GreetingResource {

    @GET
    @Produces(MediaType.TEXT_PLAIN)
    public String hello() {
        return "Hello RESTEasy";
    }
}

这是一个非常简单的 REST 端点，向 "/hello" 端点发送请求返回"Hello RESTEasy"。

与传统 JAX-RS的区别

  • 使用 Quarkus 不需要创建 Application 类 ，可以创建，但是没有必要。

  • 在多个请求中只会有一个资源实例。

  • 您可以通过配置改变只有一个资源实例的现状，通过 *Scoped 注解 (ApplicationScoped, RequestScoped 等) 来进行配置。

5. 使用注入

Quarkus 中的依赖注入基于 ArC，它是为 Quarkus 架构量身定制的基于 CDI 的依赖注入解决方案。

如果您不熟悉 CDI，那么我们建议您阅读 CDI 简介 指南。

Quarkus 仅实现了 CDI 功能的一个子集，并附带非标准功能和特定 APIS，您可以在 上下文和依赖注入指南 中了解更多信息。

ArC 是 quarkus-resteasy 的依赖项，在项目创建时已经加入到依赖集合中。

让我们修改应用程序并添加一个 bean。

创建 src/main/java/org/acme/GreetingService.java 文件，文件内容如下

package org.acme;

import javax.enterprise.context.ApplicationScoped;

@ApplicationScoped
public class GreetingService {

    public String greeting(String name) {
        return "hello " + name;
    }

}

编辑 GreetingResource 类注入 GreetingService 依赖，并使用它创建一个新端点：

package org.acme;

import javax.inject.Inject;
import javax.ws.rs.GET;
import javax.ws.rs.Path;
import javax.ws.rs.Produces;
import javax.ws.rs.core.MediaType;

import org.jboss.resteasy.annotations.jaxrs.PathParam;

@Path("/hello")
public class GreetingResource {

    @Inject
    GreetingService service;

    @GET
    @Produces(MediaType.TEXT_PLAIN)
    @Path("/greeting/{name}")
    public String greeting(@PathParam String name) {
        return service.greeting(name);
    }

    @GET
    @Produces(MediaType.TEXT_PLAIN)
    public String hello() {
        return "hello";
    }
}

如果您停止了应用程序（请记住，您不必这样做，所有改动将由Quarkus实时重新加载功能自动部署），请使用以下命令重新启动应用程序：

CLI

Gradle

hello quarkus

6. 开发模式

quarkus:dev 命令会在开发模式下启动 Quarkus 应用。它可以后台编译实现并实时重新加载，这意味着当您修改Java文件、资源文件和刷新浏览器，这些更改将会自动生效。这也适用于配置属性文件等资源文件。

刷新浏览器将会触发Quarkus对工作空间的扫描，如果检测到任何更改，则重新编译 Java 文件并重新部署应用程序，然后，重新部署的应用程序会为您的请求提供服务。如果编译或部署有任何问题，错误页面会通知您。

Quarkus会监听调试端口 5005 ，如果您想在运行前连接到该端口可以在命令行中输入 -Dsuspend 参数。如果不想要调试可以使用 -Ddebug=false 命令关闭调试。

7. 测试

到目前为止一切都很好，但是以防万一，需要做好单元测试。

在生成的项目文件中，您可以看到 2 个测试依赖项：

pom.xml
<dependency>
    <groupId>io.quarkus</groupId>
    <artifactId>quarkus-junit5</artifactId>
    <scope>test</scope>
</dependency>
<dependency>
    <groupId>io.rest-assured</groupId>
    <artifactId>rest-assured</artifactId>
    <scope>test</scope>
</dependency>
build.gradle
testImplementation("io.quarkus:quarkus-junit5")
testImplementation("io.rest-assured:rest-assured")

Quarkus 支持 JUnit 5 测试。

因此，对于 Maven，必须设置 https:maven.apache.orgsurefiremaven-surefire-plugin[Surefire Maven Plugin] 的版本，因为默认版本不支持 JUnit 5：

<plugin>
    <artifactId>maven-surefire-plugin</artifactId>
    <version>${surefire-plugin.version}</version>
    <configuration>
       <systemPropertyVariables>
          <java.util.logging.manager>org.jboss.logmanager.LogManager</java.util.logging.manager>
          <maven.home>${maven.home}</maven.home>
       </systemPropertyVariables>
    </configuration>
</plugin>

我们还设置了 java.util.logging 系统属性以确保测使用正确的日志管理器和 maven.home 确保配置信息来自 {maven.home}confsettings.xml 文件中的自定义配置（如果有）。

生成的项目包含一个简单的测试。 编辑 srctestjavaorgacmeGreetingResourceTest.java 内容如下：

package org.acme;

import io.quarkus.test.junit.QuarkusTest;
import org.junit.jupiter.api.Test;

import java.util.UUID;

import static io.restassured.RestAssured.given;
import static org.hamcrest.CoreMatchers.is;

@QuarkusTest
public class GreetingResourceTest {

    @Test    (1)
    public void testHelloEndpoint() {
        given()
          .when().get("/hello")
          .then()
             .statusCode(200)    (2)
             .body(is("hello"));
    }

    @Test
    public void testGreetingEndpoint() {
        String uuid = UUID.randomUUID().toString();
        given()
          .pathParam("name", uuid)
          .when().get("/hello/greeting/{name}")
          .then()
            .statusCode(200)
            .body(is("hello " + uuid));
    }

}
1 通过使用 QuarkusTest 注解表示正在使用 Quarkus 单元测试
2 检查HTTP响应状态码和内容

这些测试使用 http:rest-assured.io[RestAssured] 库，但请随意使用您最喜欢的库。

您可以使用 Maven 命令执行测试：

./mvnw test

您也可以直接从 IDE 运行测试（确保先停止应用程序）。

默认情况下，测试将在端口 8081 上运行，以免与正在运行的应用程序冲突。我们自动配置 RestAssured 以使用此端口。如果您想使用不同的客户端，您应该使用 @TestHTTPResource 注解将被测应用的URL直接注入到测试类的一个字段中。 该字段的类型可以是 String 、 URL 或 URI。这个注解也可以被赋予一个测试路径的值。

例如，如果想测试映射到 myservlet 的 Servlet，我只需将以下内容添加到我的测试中：

@TestHTTPResource("/myservlet")
URL testUrl;

测试端口可以通过 quarkus.http.test-port 配置进行控制。 Quarkus 还创建了一个名为 test.url 的属性，在无法使用注入的情况下将其设置为基本测试 URL。

8. 使用多模块项目或外部模块

Quarkus 在构建时大量使用 https:github.comwildflyjandex[Jandex] 来发现各种类或注解。能够立即识别的应用是 CDI bean 。 因此，如果未正确配置Jandex数据， Quarkus 扩展大多数情况下无法正常运行。

由于使用 Maven 和 Gradle 插件，该索引是默认在创建 Quarkus 项目时配置完成。

但是，在使用多模块项目时，请务必阅读 Maven 或 Gradle。

如果您计划使用外部模块（例如，一个第三方库），如果您可以修改这些外部依赖，那么可以自增加 Jandex 插件或者通过 application.properties 中的 quarkus.index-dependency` 属性(这个操作在无法修改外部依赖的时候很有用)

请务必阅读 CDI 指南的 Bean Discovery 部分以获取更多信息。

在运行程序之前，请不要忘记停止热重载模式(键盘输入 CTRL+C )，否则会出现端口冲突。

10. 配置 Banner

By default when a Quarkus application starts (in regular or dev mode), it will display an ASCII art banner. The banner can be disabled by setting quarkus.banner.enabled=false in application.properties, by setting the -Dquarkus.banner.enabled=false Java System Property, or by setting the QUARKUS_BANNER_ENABLED environment variable to false. Furthermore, users can supply a custom banner by placing the banner file in src/main/resources and configuring quarkus.banner.path=name-of-file in application.properties.

默认情况下，当Quarkus应用程序启动（在常规或开发模式下）时，它将显示 ASCII 表示的Banner。您可以在 application.properties 配置文件中添加 quarkus.banner.enabled=false 属性或者在启动时配置 -Dquarkus.banner.enabled=false 或者设置系统环境变量 QUARKUS_BANNER_ENABLED 为 false 来禁用 Banner。 此外，您可以通过将 Banner 文件放在 src/main/resources 路径下，并且在 application.properties 配置文件中添加 quarkus.banner.path=name-of-file 来修改默认的 ASCII Banner。
